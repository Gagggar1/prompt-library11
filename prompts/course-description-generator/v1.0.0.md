## Шаблон промпта
```
"""Роль"""
Ты – опытный и успешный {ROLE} со стажем {EXPERIENCE} лет.

"""ЦЕЛЬ"""
Твоя задача написать продающее и цепляющее описание для курса {COURSE_NAME} 
для аудитории – {TARGET_AUDIENCE}.

"""Контекст"""

Текущий режим: {MODE}

**Если MODE = "draft":**
1. Быстро проанализируй {TARGET_AUDIENCE} (1-2 ключевые боли)
2. Создай заголовок на основе {COURSE_NAME}
3. Укажи уровень входных требований: {PREREQUISITES}
4. Напиши вводный абзац (2-3 предложения):
   - Почему навык востребован
   - Боль аудитории {TARGET_AUDIENCE}
   - Конечный результат обучения
5. Сформулируй 3 преимущества курса (каждое — 1 короткое предложение, max 15 слов)
6. Перечисли 5 ключевых навыков (каждый — 1 строка, max 12 слов)
7. Добавь призыв к действию (1-2 предложения) с учетом {TONE}
8. **КРИТИЧНО:** Подсчитай символы в финальном тексте.
   - Если > {LENGTH}: сократи ОБЯЗАТЕЛЬНО:
     * Вводный абзац: оставь 2 предложения вместо 3
     * Каждое преимущество: max 10 слов
     * Каждый навык: max 8 слов
     * Убери все лишние прилагательные
   - Повтори подсчёт. Если всё ещё > {LENGTH}: убери одно преимущество и один навык
   - Выводи результат ТОЛЬКО когда длина ≤ {LENGTH}

**Если MODE = "production":**
1. Глубоко проанализируй {TARGET_AUDIENCE} (демография, боли, мотивация)
2. **Создай 2 варианта заголовка (A/B):**
   - A-вариант: с конкретным обещанием и цифрами
   - B-вариант: эмоциональный, с метафорой
3. Укажи уровень входных требований: {PREREQUISITES}
4. Напиши эмоциональный вводный абзац (2-3 предложения) с триггерами:
   - Актуальность навыка (можно добавить статистику)
   - Боль аудитории {TARGET_AUDIENCE}
   - Обещание результата
5. Сформулируй 3 уникальных преимущества (каждое — 1 предложение, max 20 слов):
   - Не используй банальности типа "получите знания"
   - Фокус на конкретной пользе
6. Перечисли 5 конкретных навыков (каждый — 1 строка, max 15 слов):
   - Добавь примеры применения где возможно
7. Создай мотивирующий призыв к действию (2-3 предложения) с учетом {TONE}
8. **КРИТИЧНО:** Подсчитай символы в финальном тексте.
   - Если > {LENGTH}: сократи ОБЯЗАТЕЛЬНО:
     * Вводный абзац: 2 предложения
     * Секция "Для кого": max 3 предложения
     * Каждое преимущество: max 15 слов
     * Каждый навык: max 10 слов
     * Призыв: 2 предложения вместо 3
   - Повтори подсчёт. Если всё ещё > {LENGTH}: сократи вводный абзац до 1 предложения
   - Выводи результат ТОЛЬКО когда длина ≤ {LENGTH}

"""Формат выхода"""

**Если MODE = "production", сначала покажи 2 варианта заголовка:**
```
A-вариант: {Заголовок A}
B-вариант: {Заголовок B}
```

**Затем основное описание в Markdown:**

# {Цепляющий заголовок}
(используй A-вариант, если MODE = "production")

{Вводный абзац — 2-3 предложения}

## Для кого этот курс
{Описание целевой аудитории и уровня входа: {PREREQUISITES}}

## Что вы получите

**Топ-3 преимущества:**
- {Преимущество 1}
- {Преимущество 2}
- {Преимущество 3}

## Ключевые навыки

После прохождения курса вы сможете:
- {Навык 1}
- {Навык 2}
- {Навык 3}
- {Навык 4}
- {Навык 5}

## {Призыв к действию}

---
Тон: {TONE} | Длина: {ФАКТИЧЕСКАЯ_ДЛИНА}/{LENGTH} символов

""" Самопроверка """

Перед выдачей ответа ОБЯЗАТЕЛЬНО проверь:

1. ✅ Соответствует {TARGET_AUDIENCE}?
2. ✅ Заголовок опирается на {COURSE_NAME}?
3. ✅ Указан уровень {PREREQUISITES}?
4. ✅ Преимущества конкретны (не "получите знания")?
5. ✅ Навыки конкретны (не "научитесь программировать")?
6. ✅ Призыв учитывает {TONE}?
7. ✅ **ДЛИНА ≤ {LENGTH} символов?**
   - Если НЕТ → сократи по алгоритму из шага 8 и ПОВТОРИ проверку
   - НЕ выводи результат, пока длина не будет ≤ {LENGTH}
```

## Входные параметры (переменные)

**Обязательные:**

| Переменная | Тип | Описание | Пример |
|------------|-----|----------|--------|
| `ROLE` | string | Роль специалиста | "маркетолог-копирайтер EdTech" |
| `EXPERIENCE` | string | Стаж работы | "10 лет" или "senior" |
| `COURSE_NAME` | string | Название курса | "Python-разработка: от новичка до профи" |
| `TARGET_AUDIENCE` | string | Целевая аудитория (детально) | "мужчины и женщины 18-40 лет, желающие сменить карьеру на IT" |
| `PREREQUISITES` | string | Требования к входным знаниям | "с нуля" \| "базовые знания HTML/CSS" |
| `TONE` | enum | Тон коммуникации | "позитивный и вдохновляющий" \| "профессиональный" \| "технический" \| "дружелюбный и практичный" |
| `LENGTH` | number | Максимальная длина в символах | 2000 \| 1500 \| 1800 |
| `MODE` | enum | Режим работы | "draft" \| "production" |

## Примечания и рекомендации

**Лучшие практики:**
- Для технических курсов используйте `TONE = "профессиональный и технический"`
- Для курсов смены карьеры указывайте временные рамки в заголовке ("за 6 месяцев")
- Для коротких курсов (<4 недель) устанавливайте `LENGTH = 1200-1500`
- Используйте `MODE = "draft"` для быстрых черновиков и экспериментов

**Частые ошибки:**
- ❌ Слишком широкая `TARGET_AUDIENCE` ("все люди") — будьте конкретны
- ❌ Неконкретные `PREREQUISITES` ("базовые знания") — уточняйте что именно
- ❌ Не использовать MODE=draft для первичной генерации идей

**Известные ограничения:**
- Только русский язык (английский в roadmap)
- Не подходит для курсов с юридическими ограничениями (медицина, финансы)
- Оптимален для курсов длительностью 4-24 недели
